# Автоматическое определение региона пользователя

## Описание

Реализована функциональность автоматического определения региона пользователя при авторизации. Регион определяется только один раз после авторизации, далее пользователь может самостоятельно выбирать нужный регион.

## Как это работает

### 1. Автоматическое определение при авторизации

Функция `detectUserRegion()` автоматически вызывается в следующих случаях:

- **После успешного входа** (`loginUser`)
- **После успешной аутентификации** (`authenticate`)
- **После верификации SMS-кода** (`verifyCode`)
- **При загрузке профиля пользователя** (`getUser`)
- **При загрузке главной страницы** (`HomeModule`)
- **При загрузке ленты постов** (`HomePostsModule`)
- **При загрузке профиля** (`ProfileModule`)

### 2. Логика определения региона

1. **Проверка существующего региона**: Если регион уже установлен, функция завершается
2. **Получение геолокации по IP**: Используется API для определения страны по IP-адресу
3. **Поиск региона в базе**: Ищется соответствующая страна в списке доступных регионов
4. **Установка региона**: Если страна найдена, она устанавливается как текущий регион
5. **Уведомление пользователя**: Показывается сообщение об автоматически определенном регионе

### 3. Сохранение в localStorage

Определенный регион автоматически сохраняется в localStorage через Redux store subscription и восстанавливается при перезагрузке страницы.

## API endpoints

- **Геолокация по IP**: `Pathes.Common.myLocation`
- **Список стран и городов**: `Pathes.Common.citiesAndCountries`

## Структура данных региона

```javascript
{
  type: "countries",
  code: "KG", // Код страны
  name: "Kyrgyzstan", // Название страны
  flag: "/media/flags-iso/KG.png", // Флаг страны
  currency: {
    code: "KGS", // Код валюты
    name: "Kyrgystani Som" // Название валюты
  }
}
```

## Особенности

1. **Однократное определение**: Регион определяется только если он не установлен
2. **Автоматическое сохранение**: Регион сохраняется в localStorage
3. **Уведомления**: Пользователь получает уведомление об автоматически определенном регионе
4. **Возможность изменения**: Пользователь может изменить регион в любое время
5. **Обработка ошибок**: Если определение не удалось, пользователь может выбрать регион вручную

## Логирование

В консоли браузера отображаются сообщения:

- Успешное определение: `"Автоматически определен регион: [Название страны]"`
- Ошибки: `"Не удалось автоматически определить регион"`

## Безопасность

- Функция работает только для авторизованных пользователей
- Геолокация происходит только по IP-адресу
- Не используются GPS или другие методы определения местоположения
- Данные не передаются третьим лицам
